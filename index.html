Secure 360 – Azure Deployment Blueprint

Principles

  • Zero trust by default: private endpoints, managed identities, RBAC.
  • Separation of concerns: isolate environments, workloads, and data paths.
  • Event-first design: Functions + queues, not long-running services.
  • Data contract: normalize inputs into curated schema.
  • Operational clarity: observable, testable, IaC deployable.
  • Cost awareness: serverless where possible, hot vs cold storage.

Network & Landing Zone

  • Subscriptions: Management, Data, App, Analytics.
  • Resource groups: rg-network, rg-identity, rg-data, rg-app, rg-ops.
  • Hub-and-spoke VNet with Firewall + Private DNS.
  • Private Endpoints: SQL, Storage, Key Vault, Log Analytics.
  • Regions: South Africa North primary, South Africa West failover.

Identity & Access

  • Microsoft Entra ID app registration with roles: Viewer, Analyst, Admin.
  • Group-based assignments drive Power BI RLS.
  • Managed identities for Functions, Dataflows.
  • Conditional Access with MFA, no legacy auth.
  • Secrets in Key Vault, purge protection enabled.

Data Architecture

  • Ingest: Functions pull Defender & Graph → Blob Storage.
  • Normalize: Functions → curated SQL tables (Resources, Alerts, Recommendations, Identities, Incidents).
  • Enrich: Ownership mapping, deduplication, incidentization.
  • Retention: Blob 30–90 days, SQL 12 months with roll-ups.

Applications & Automation

  • Azure Functions: Timer + HTTP triggers for ingestion and remediation.
  • Automated remediation: Teams adaptive cards, ticket webhooks, resource tagging.
  • Power BI dashboards with RLS for posture, alerts, identity hygiene.

CI/CD & Infrastructure as Code

  • Environments: dev, test, prod with separate subscriptions.
  • Bicep/Terraform for VNet, Functions, SQL, Storage, Key Vault, App Insights.
  • GitHub Actions/Azure DevOps pipelines with OIDC, policy guardrails.
  • Blue/green deployments for Functions, migration scripts for SQL.

Observability & Resilience

  • App Insights + Log Analytics for telemetry and auditing.
  • Azure Monitor alerts: ingestion failures, queue backlogs, Function errors, secure score drops.
  • Resilience: PITR backups, storage lifecycle, failover to SA West.
  • Runbooks: ingestion retry, break-glass access, schema drift handling.

Cost & Scaling

  • Serverless Functions (consumption plan).
  • SQL: General Purpose → Hyperscale as needed.
  • Storage: Hot → Cool → Archive tiers.
  • Power BI: Pro for dev/test, Premium for scale.
  • Scaling path: shard by subscription, archive yearly, multi-tenant schemas.
